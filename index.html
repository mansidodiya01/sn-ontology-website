Surya Namaskar Ontology

Semantic Modeling | OWL | SPARQL | Knowledge Representation

This ontology models Surya Namaskar as poses, sequence steps, and variants. It supports structured querying of sequence order, repeated steps, inverse relationships, support type, mantra, chakra, and shared poses across variants.

The ontology is designed to represent multiple Surya Namaskar traditions in a unified structure so that they can be compared and queried systematically.

Ontology Overview

Classes

  • Pose – Represents a yoga pose identity.
  • SequenceStep – Represents one occurrence of a pose in a sequence.
  • Variant – Represents a Surya Namaskar tradition or sequence variant.

Object Properties

  • hasPose – Links a sequence step to the pose performed at that step.
  • belongsToVariant – Links a sequence step to its variant.
  • hasNextStep – Links a step to the next step.
  • hasPreviousStep – Links a step to the previous step.
  • repeatsStep – Links repeated or mirrored steps.
  • hasInverseStep – Links inverse or opposite steps.
  • samePoseAs – Links equivalent or normalized pose identities.

Datatype Properties

  • stepNumber – Position of a step in a sequence.
  • hasMantra – Mantra associated with a step.
  • hasChakra – Chakra associated with a step.
  • hasSupportType – Support/body position type for a step.
  • isStandingOnTwoFeet – Indicates whether a pose is performed on two feet.
  • isStandingOnAtLeastOneFoot – Indicates whether at least one foot is used for support.
  • hasCYPPage – CYP reference page for a pose.
  • hasAlternateName – Alternate name of a pose.

SPARQL Queries

C1: What are the poses in Surya Namaskar?

(a) List all pose identities

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?pose ?label
WHERE {
  ?pose rdf:type :Pose ;
        rdfs:label ?label .
}
ORDER BY ?label

(b) Count all pose identities

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#>

SELECT (COUNT(?pose) AS ?poseCount)
WHERE {
  ?pose rdf:type :Pose .
}

C2: How many poses are in Base Surya Namaskar and what are they?

(a) Count poses in Base SN

PREFIX : <http://example.org/suryanamaskar#>

SELECT (COUNT(?step) AS ?count)
WHERE {
  ?step :belongsToVariant :BaseSN .
}

(b) List poses in Base SN

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?stepNo ?poseLabel
WHERE {
  ?step :belongsToVariant :BaseSN ;
        :stepNumber ?stepNo ;
        :hasPose ?pose .
  ?pose rdfs:label ?poseLabel .
}
ORDER BY ?stepNo

C4: Which poses are repeated in Surya Namaskar?

(a) Repeated steps in Base SN

PREFIX : <http://example.org/suryanamaskar#>

SELECT ?step1 ?n1 ?step2 ?n2
WHERE {
  ?step1 :belongsToVariant :BaseSN ;
         :repeatsStep ?step2 ;
         :stepNumber ?n1 .
  ?step2 :stepNumber ?n2 .
  FILTER(?n1 < ?n2)
}
ORDER BY ?n1

(b) Repeated poses with labels

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?n1 ?poseLabel1 ?n2 ?poseLabel2
WHERE {
  ?step1 :belongsToVariant :BaseSN ;
         :repeatsStep ?step2 ;
         :stepNumber ?n1 ;
         :hasPose ?pose1 .
  ?step2 :stepNumber ?n2 ;
         :hasPose ?pose2 .
  ?pose1 rdfs:label ?poseLabel1 .
  ?pose2 rdfs:label ?poseLabel2 .
  FILTER(?n1 < ?n2)
}
ORDER BY ?n1

C7: Which poses are common across variants?

(a) Poses appearing in more than one variant

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?poseLabel (COUNT(DISTINCT ?variant) AS ?variantCount)
WHERE {
  ?step :belongsToVariant ?variant ;
        :hasPose ?pose .
  ?pose rdfs:label ?poseLabel .
}
GROUP BY ?poseLabel
HAVING (COUNT(DISTINCT ?variant) > 1)
ORDER BY DESC(?variantCount) ?poseLabel

(b) Show variants for each pose

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?poseLabel ?variantLabel
WHERE {
  ?step :belongsToVariant ?variant ;
        :hasPose ?pose .
  ?pose rdfs:label ?poseLabel .
  ?variant rdfs:label ?variantLabel .
}
ORDER BY ?poseLabel ?variantLabel

C8: What is the sequence order of poses?

(a) Next pose (example: Step 4)

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?currentNo ?currentPoseLabel ?nextNo ?nextPoseLabel
WHERE {
  :BaseSN_Step_04 :stepNumber ?currentNo ;
                  :hasPose ?currentPose ;
                  :hasNextStep ?nextStep .
  ?currentPose rdfs:label ?currentPoseLabel .
  ?nextStep :stepNumber ?nextNo ;
            :hasPose ?nextPose .
  ?nextPose rdfs:label ?nextPoseLabel .
}

(b) Previous pose (example: Step 4)

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?prevNo ?prevPoseLabel ?currentNo ?currentPoseLabel
WHERE {
  :BaseSN_Step_04 :stepNumber ?currentNo ;
                  :hasPose ?currentPose ;
                  :hasPreviousStep ?prevStep .
  ?currentPose rdfs:label ?currentPoseLabel .
  ?prevStep :stepNumber ?prevNo ;
            :hasPose ?prevPose .
  ?prevPose rdfs:label ?prevPoseLabel .
}

(c) Full sequence order in Base SN

PREFIX : <http://example.org/suryanamaskar#>
PREFIX rdfs: <http://www.w3.org/2000/01/rdf-schema#>

SELECT ?currentNo ?currentPoseLabel ?nextNo ?nextPoseLabel
WHERE {
  ?step :belongsToVariant :BaseSN ;
        :stepNumber ?currentNo ;
        :hasPose ?currentPose ;
        :hasNextStep ?nextStep .
  ?currentPose rdfs:label ?currentPoseLabel .
  ?nextStep :stepNumber ?nextNo ;
            :hasPose ?nextPose .
  ?nextPose rdfs:label ?nextPoseLabel .
}
ORDER BY ?currentNo
